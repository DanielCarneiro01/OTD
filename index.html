O que é

Quando se está trabalhando com uma interface remota, como uma Fachada Remota, cada chamada possui um custo muito alto. Para tentar reduzir o número de chamadas, o que significa que você precisa transferir mais dados em cada chamada. Uma maneira de fazer isso é utilizando muitos parâmetros.

A solução é criar um Objeto de Transferência de Dados, que pode armazenar todos os dados de uma chamada. Ele precisa ser serializavel para passar pela conexão. Normalmente um montador é usado no lado servidor para transferir dados entre o OTD e o objeto de domínio.

Como Funciona.

Sempre que um objeto remoto precisa de alguns dados, pede por uma Objeto de Transferência de Dados apropriados. O OTD geralmente transporta muito mais dados do que um objeto remoto solicitou, mas deve transportar todos os dados que um objeto remoto precisara por algum tempo. Devido ao custo de latência é preferível errar enviando dados de mais do que ter que fazer diversas chamadas. Um único OTD normalmente contém mais do que apenas um único objeto do servidor. Ele agrega dados de todos os objetos do servidor que o objeto remoto provavelmente ira querer.
